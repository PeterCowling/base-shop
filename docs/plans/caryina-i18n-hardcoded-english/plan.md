# Caryina Chrome i18n Plan

## Summary

All four Caryina chrome components (Header, SiteFooter, ConsentBanner, ShippingReturnsTrustBlock) are already locale-wired via `getChromeContent(lang)`. The bug is that `CHROME_EN_DEFAULTS` and the JSON chrome section both have only English values — no German or Italian translations exist anywhere. The fix is: (1) rename the constant to `CHROME_DEFAULTS`, add 50 new translation values (25 strings × 2 locales), and atomically remove the `chrome` key from `site-content.generated.json` so the enriched code constant becomes the active source; (2) add a focused unit test confirming locale-correct output. No component changes. No schema changes. Two tasks.

## Active tasks

- [x] TASK-01: Add DE/IT translations to `CHROME_DEFAULTS` and remove JSON chrome
- [x] TASK-02: Add `contentPacket.test.ts` locale resolution unit tests

## Goals

- Add DE/IT translations for all 25 chrome strings across 5 groups (header, footer, consent, trust, notifyMe)
- Make `CHROME_DEFAULTS` the authoritative and durable source of truth for chrome translations
- Remove the `chrome` key from the generated JSON to eliminate materializer durability risk
- Add unit test coverage for `getChromeContent` locale resolution

## Non-goals

- Translating product copy, policy body text, or SEO content
- Adding new chrome components or changing chrome structure
- Internationalising admin routes
- Fixing the `productPage.trustStrip` materializer durability risk (separate concern, pre-existing)

## Constraints & Assumptions

- Constraints:
  - Materializer (`materialize-site-content-payload.ts`) must not require changes
  - TASK-01 code change + JSON chrome removal must be a single atomic commit (JSON chrome wins over code if only one is done)
  - Translations must be natural DE/IT — not machine-literal
- Assumptions:
  - `LocalizedText` type already has `de?` and `it?` optional keys — no type changes needed
  - `cachedPayload` in `readPayload()` is process-level — no runtime cache invalidation required for deploy

## Inherited Outcome Contract

- **Why:** German and Italian visitors get a mixed-language experience — translated product pages but English navigation, consent banner, footer links, and trust strip. This undermines trust, especially on legal pages where language matters.
- **Intended Outcome Type:** operational
- **Intended Outcome Statement:** All persistent UI chrome (header, footer, consent banner, trust strip) displays in the visitor's selected language
- **Source:** operator
