# Analysis Module: Mixed Track

Use for `Execution-Track: mixed`.

## Objective

Produce one coherent recommended approach across technical and business dimensions without leaking task planning into the analysis stage.

## Required Outputs

- Shared option set covering both tracks
- Explicit tradeoffs across code and business execution
- Chosen cross-track approach
- Planning handoff notes for both tracks

## Comparison Heuristics

- Compare technical feasibility and business validation together.
- Reject options that optimize one track by creating hidden burden in the other.
- Make dependencies between technical and business work explicit so `/lp-do-plan` can sequence them cleanly.
