II. Curation Rules

Rule Description
Genuine outputs only Research, plans, baselines, forecasts, scorecards, decision logs, briefs, feasibility studies, signal reviews.
No prompts or templates Files whose primary purpose is to feed a process are excluded (e.g. *-prompt.user.md, *-request-prompt.user.md).
Pointer-first Where a latest.user.md pointer exists for market-research or site-upgrades, the pointer appears — not the dated source it references.
Latest version only When multiple numbered versions of a document exist (e.g. forecast v1 + v2), only the latest is shown.
No exec summaries Always derivative of their parent document. Excluded.
No .agent.md variants Agent-facing pipeline files. Excluded.
Content test, not suffix test Some genuine outputs use plain .md (auto-generated analyses, signal reviews). The test is frontmatter Type: or evident content, not the filename suffix.

VI. Exclusions

Excluded file categories

*-exec-summary.* *.agent.md *.user.html *-prompt.user.md *-request-prompt.* index.user.md *-sources.md superseded versions
Exec summaries are derivative of their parent forecast and excluded across all three businesses. Agent variants (.agent.md) and rendered HTML copies (.user.html) are excluded in favour of the canonical .user.md source. Prompt and request-prompt files are inputs to processes, not outputs of them. Navigation indexes (index.user.md) and source lists (*-sources.md) are supporting material, not decision-grade artifacts. Superseded forecast versions: HEAD and PET v1 forecasts are excluded; only v2 is shown.

Suffix note: .user.md is the conventional suffix for operator-facing docs but is not the inclusion criterion. room-pricing-analysis.md (BRIK, Type: Analysis-Output) and signal-review-*.md (BRIK, Type: Signal-Review) use plain .md and are included on content grounds.
