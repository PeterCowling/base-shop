Free shipping threshold is hardcoded — not registered as standing intelligence
R9
During the HBAG pricing uplift build, launchMerchandising.ts was found to contain a hardcoded free-shipping threshold (€150 → updated to €180). This value is price-sensitive and drifts silently whenever prices change. It is not currently registered as a standing artifact, meaning future price changes will not automatically surface it as a downstream touchpoint.

Decision required: Extract threshold into a pricing policy standing artifact (e.g. docs/business-os/startup-baselines/HBAG-pricing-policy.md) and register it in the Update Triggers table in two-layer-model.md so it is monitored alongside the pricing doc.
Recommended action
Create HBAG-pricing-policy.md with the threshold value and its rationale. Register in Update Triggers. Record decision in results-review.user.md under ## Standing Expansion.
